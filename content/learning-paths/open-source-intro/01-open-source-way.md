## Learning goals

- Describe what the open source way means beyond code.
- Explain five core principles used in open collaboration.
- Recognize how these principles improve team outcomes.

## Core ideas

The open source way is a practical collaboration model that can be used in software, education, public sector work, and organizations.

Core principles include:

- **Transparency**: people can see information and decisions.
- **Collaboration**: contributors build on each other’s work.
- **Release early and often**: iterative delivery creates faster learning.
- **Inclusive meritocracy**: good ideas can come from anywhere.
- **Community**: shared goals and values guide action.

## Why it matters

Teams working this way usually learn faster, reduce duplicate effort, and build stronger trust because decisions and progress are visible.

## Practice

Pick one current project and list one change you could make this week for each principle.
